Installation

Getting started

There's one manual step: burn the ISO to a USB drive. Everything after that is automatic. The platform has two modes — live desktop and provisioning tool — both start from the same USB key.

Mode 1 — Live Desktop

Plug the USB into any machine and select it as the boot device. A full Debian 13 GNOME desktop loads directly from the drive. Nothing is written to the host machine's disks.

This mode is completely ephemeral. Everything that happens during the session — credentials generated, configurations applied, secrets created — exists only in RAM. When the machine powers off or the USB is removed, it's all gone. This makes it safe to use on any hardware, in any environment, without leaving a trace.

The Web UI opens automatically at boot. From there you can explore the platform, test configurations, or choose to install permanently onto the host machine. The install process takes about 90 seconds.

Write to USB

On Windows: use Rufus or Balena Etcher.

On Linux or macOS: dd if=debz.iso of=/dev/sdX bs=4M status=progress

On a VM: attach the ISO directly as a virtual disk — no USB needed.

Mode 2 — Provisioning a Node

Boot the ISO on a machine you want to configure permanently. The Web UI guides you through a short wizard — pick a role, pick a disk, confirm. The platform then:

  • Partitions the disk and creates a ZFS storage pool
  • Installs Debian 13 with the correct packages for the selected role
  • Configures WireGuard mesh networking
  • Sets up automatic snapshots before every system update
  • Optionally enrolls a Secure Boot signing key

When it finishes, the machine reboots into a clean, permanent Debian 13 system. From this point it can be managed directly through the Web UI, handed off to any existing IaC toolchain (Ansible, Terraform, Salt — it doesn't care), or used as a golden image to clone from.

Node roles

Each node has a role that determines what gets installed. Roles are not restrictive — they're a starting point. You can always add to them.

  • Desktop — GNOME desktop, suitable for workstations or developer machines
  • Server — headless, SSH, minimal footprint
  • KVM — KVM hypervisor, ready to run virtual machines
  • Kubernetes — Kubernetes node, joins the cluster automatically over WireGuard
  • Storage — dedicated ZFS storage host with replication support

Deploying a Cluster

Provision one node, configure it the way you want, then use the golden image pipeline to clone it. A golden image is a complete snapshot of a fully installed and configured node. Clone it once, clone it fifty times — each copy gets its own unique identity, hostname, and WireGuard keys automatically.

Clones can be deployed to any Proxmox host, AWS, Azure, GCP, or bare metal — from the same Web UI, in a single operation. The WireGuard mesh connects them automatically as long as the mesh ports are reachable.

The full technical reference for golden images, answer files, and cluster bootstrap is on GitHub.
