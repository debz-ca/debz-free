deploy.sh Reference

All builds, deployments, and image operations are orchestrated through deploy.sh. This page covers every command, environment variable, and common usage patterns.

Host requirements

  • docker or podman — all build work runs inside a container
  • git — for commit-stamping builds
  • rsync — for copying build artifacts
  • qemu-system-x86 + qemu-utils — for golden, stamp
  • awscli — for aws-ami

Configuration: debz.env

Place persistent overrides in debz.env at the repository root. This file is sourced by deploy.sh if it exists. Never commit it — add it to .gitignore. A template is provided as debz.env.example. 

# debz.env — copy from debz.env.example and customize
PROFILE=desktop
BUILDER_IMAGE=debz-live-builder:latest
OUTPUT_DIR=live-build/output
PROXMOX_HOST=proxmox-01.debz.local
VMID=100
VM_NAME=debz-desktop-01
Commands

build / iso

Builds the debz live ISO inside a Docker container using live-build. iso is an alias for build.

# Build default desktop ISO
./deploy.sh build

# Build server ISO
./deploy.sh build PROFILE=server

# Build for ARM64 (cross-compile)
./deploy.sh build ARCH=arm64

# Custom output directory
./deploy.sh iso OUTPUT_DIR=/mnt/nas/debz-isos/
VariableDefaultDescription
PROFILEdesktopdesktop | server
ARCHamd64amd64 | arm64
BUILDER_IMAGEdebz-live-builder:latestBuilder container image name
OUTPUT_DIRlive-build/outputISO output directory
LOG_DIRlive-build/logsBuild log directory

deploy

Deploy a built ISO to a Proxmox host: uploads the ISO, creates a VM, and optionally starts it.

# Deploy to Proxmox with defaults from debz.env
./deploy.sh deploy

# Specific VM configuration
./deploy.sh deploy \
  PROXMOX_HOST=proxmox.debz.local \
  VMID=101 \
  VM_NAME=debz-server-01 \
  VM_MEMORY=8192 \
  VM_CORES=4 \
  VM_DISK_GB=60

# With TPM + Secure Boot pre-configured
./deploy.sh deploy VM_SECURE_BOOT=1 VM_TPM=1

# Burn ISO to USB at deploy time
./deploy.sh deploy USB_BURN_ON_DEPLOY=1 USB_DEVICE=/dev/sdb
VariableDefaultDescription
PROXMOX_HOSTrequiredProxmox VE hostname or IP
VMID100Proxmox VM ID
VM_NAMEdebz-nodeVM display name
VM_MEMORY4096RAM in MB
VM_CORES4vCPU count
VM_DISK_GB40Primary disk size in GB
VM_EXTRA_DISKS0Number of additional data disks (for ZFS topology testing)
VM_EXTRA_DISK_GB40Size of each extra disk in GB
VM_BRIDGEvmbr0Proxmox network bridge
VM_SECURE_BOOT01 = enable Secure Boot on EFI disk
VM_TPM01 = attach virtual TPM 2.0
PROXMOX_DATA_STORElocal-lvmProxmox storage pool for VM disks
USB_BURN_ON_DEPLOY01 = also write ISO to USB_DEVICE
USB_DEVICEBlock device path for USB burn

full

Runs clean + build + deploy in sequence.

./deploy.sh full PROFILE=server PROXMOX_HOST=proxmox.debz.local VMID=102

golden

Builds a golden image for a template using headless QEMU. See Golden Images for full detail.

./deploy.sh golden TEMPLATE=kvm
./deploy.sh golden TEMPLATE=storage GOLDEN_SIZE=80G
./deploy.sh golden TEMPLATE=vdi GOLDEN_ISO=./live-build/output/debz-trixie-amd64-server.iso
VariableDefaultDescription
TEMPLATErequiredmaster | kvm | storage | vdi
GOLDEN_SIZE40Gqcow2 disk size
GOLDEN_FORMATqcow2qcow2 | raw
GOLDEN_ISOlive-build/output/latest.isodebz ISO to install from
GOLDEN_OUTPUT/var/lib/debz/golden/Output directory

stamp

Creates CoW clones of a golden image with injected per-node identity.

./deploy.sh stamp TEMPLATE=kvm STAMP_COUNT=8 STAMP_BASE_HOSTNAME=kvm STAMP_BASE_INDEX=2
./deploy.sh stamp TEMPLATE=storage STAMP_COUNT=2 STAMP_OUTPUT_DIR=/mnt/nas/clones/
VariableDefaultDescription
TEMPLATErequiredTemplate to stamp from
STAMP_COUNT1Number of clones to create
STAMP_BASE_HOSTNAMETEMPLATEHostname prefix (clones get -NN suffix)
STAMP_BASE_INDEX2Node index of first clone
STAMP_OUTPUT_DIR/var/lib/debz/clones/Clone output directory

spawn

Boots stamped clones as libvirt VMs or Proxmox VMs.

# Spawn all stamped KVM clones on local libvirt
./deploy.sh spawn TEMPLATE=kvm

# Spawn on Proxmox
./deploy.sh spawn TEMPLATE=storage PROXMOX_HOST=proxmox-01.debz.local

burn

Write the built ISO directly to a USB drive.

./deploy.sh burn USB_DEVICE=/dev/sdb
./deploy.sh burn USB_DEVICE=/dev/sdb PROFILE=server

clean

Remove build artifacts: live-build/output/, live-build/work/, and cached container layers.

./deploy.sh clean

validate

Run shellcheck on all .sh and .hook.chroot files (excluding legacy/), verify required directories and files exist, check scripts are executable, and validate profile YAML.

./deploy.sh validate
# or
make validate

builder-image

Build the builder/Dockerfile into the debz-live-builder:latest Docker image used by all build commands.

./deploy.sh builder-image
./deploy.sh builder-image BUILDER_IMAGE=my-registry.io/debz-builder:v2

vmdk / rootfs / firecracker

Convert a built ISO into alternative formats for specific deployment targets.

# VMDK for VMware / VirtualBox
./deploy.sh vmdk

# Compressed rootfs tarball
./deploy.sh rootfs

# Firecracker-compatible kernel + initrd + rootfs
./deploy.sh firecracker

aws-ami

Upload the debz rootfs to S3 and register it as an EC2 AMI.

./deploy.sh aws-ami \
  AWS_BUCKET=my-debz-amis \
  AWS_REGION=us-east-1 \
  AWS_AMI_NAME=debz-trixie-server-$(date +%Y%m%d)
VariableDefaultDescription
AWS_BUCKETrequiredS3 bucket for upload
AWS_REGIONus-east-1AWS region
AWS_AMI_NAMEdebz-trixie-amd64AMI name (must be unique in region)

release

Package built ISOs with SHA256 checksums into a release directory ready for upload.

./deploy.sh release OUTPUT_DIR=/mnt/releases/debz-2026-03-16/
Example profiles

Quick local test

# Build server ISO and verify it with validate
./deploy.sh validate
./deploy.sh build PROFILE=server

Desktop + Proxmox + USB

# debz.env for a desktop workstation deployed to Proxmox
PROFILE=desktop
PROXMOX_HOST=pve.debz.local
VMID=200
VM_NAME=debz-desktop-dev
VM_MEMORY=8192
VM_CORES=8
VM_DISK_GB=100
VM_SECURE_BOOT=1
VM_TPM=1
USB_DEVICE=/dev/sdc

./deploy.sh full
./deploy.sh burn   # also burn the ISO to /dev/sdc

All 4 golden images

for template in master kvm storage vdi; do
  ./deploy.sh golden TEMPLATE=$template GOLDEN_SIZE=50G
done

8-node KVM stamp

# Build once, stamp 8 clones starting at index 2
./deploy.sh golden TEMPLATE=kvm GOLDEN_SIZE=60G
./deploy.sh stamp  TEMPLATE=kvm \
  STAMP_COUNT=8 \
  STAMP_BASE_HOSTNAME=kvm \
  STAMP_BASE_INDEX=2

# Spawn all 8 on Proxmox
./deploy.sh spawn TEMPLATE=kvm PROXMOX_HOST=pve.debz.local

AWS AMI server build

# Clean build, convert to rootfs, upload as AMI
./deploy.sh clean
./deploy.sh build PROFILE=server ARCH=amd64
./deploy.sh aws-ami \
  AWS_BUCKET=my-debz-amis \
  AWS_REGION=ca-central-1 \
  AWS_AMI_NAME=debz-trixie-server-$(date +%Y%m%d)
